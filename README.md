---

### ШАГ 1: Создаем правильный `.gitignore`
Это **самое важное**. Если ты зальешь на GitHub папки `node_modules` или `venv`, проект будет весить гигабайты и у друга всё сломается.

В самой корневой папке проекта (там, где лежит `docker-compose.yml`) создай файл с названием **`.gitignore`** (обязательно с точкой в начале) и вставь туда это:

```text
# Базы данных и кэш
pgdata/
redis_data/
uploads/

# Python
backend/venv/
backend/__pycache__/
*.pyc
*.pyo
*.pyd
.pytest_cache/

# Node.js / Vue
frontend/node_modules/
frontend/dist/
frontend/.npm/

# Среды и ключи
.env
.env.local

# Системные файлы
.DS_Store
Thumbs.db
```

---

### ШАГ 2: Создаем `README.md` (Инструкция для друга)
Чтобы друг (и комиссия на защите диплома) понимали, что это за проект и как его запустить, в корневой папке создай файл **`README.md`**.

Вставь в него этот текст (друг будет читать его на GitHub):

```markdown
# Job Parser & Recommender System

Проект представляет собой рекрутинговый портал с функционалом парсинга вакансий и рекомендательной системой на основе ML.

## 🚀 Как запустить проект (для разработчиков)

Для запуска вам понадобится только установленный **Docker Desktop** и **Git**.

### 1. Клонирование репозитория
```bash
git clone <СЮДА_ВСТАВИТЬ_ССЫЛКУ_НА_GITHUB_РЕПОЗИТОРИЙ>
cd job_parser
```

### 2. Запуск через Docker
Находясь в корневой папке проекта, выполните команду:
```bash
docker-compose up -d --build
```

### 3. Готово!
Докер автоматически скачает базы данных, установит все зависимости Python и Node.js.
Проект будет доступен по адресам:
* **Frontend (САМ САЙТ):** [http://localhost:5173](http://localhost:5173)
* **Backend API (БЭКЕНД с апи-запросами):** [http://localhost:8000/docs](http://localhost:8000/docs)

*Примечание: При первом запуске скачивание образов (PostgreSQL, Redis, Python, Node) может занять 3-5 минут в зависимости от скорости интернета.*

## 🛠 Архитектура (Стек)
* **Frontend:** Vue 3 (Composition API), Pinia, Vue Router, Tailwind/Чистый CSS.
* **Backend:** Python, FastAPI, SQLAlchemy, PostgreSQL (+ pgvector), Redis.
```
